# 交通标志检测系统 - 研究内容调整版

## 📋 研究内容（根据实际实现调整）

### 1. 数据集构建与模型优化
基于 **CCTSDB2021**（中国交通标志检测基准）数据集进行模型训练。采集现实场景中的交通标志样本，针对标志检测在复杂背景、不同光照和距离条件下的挑战，实施数据增强和模型微调策略。

- **自研模型**：`bestvvv.pt` - 专门针对中国交通标志的检测模型
- **mAP@50 指标**：≥ 0.94（在测试集上验证）
- **输入分辨率**：640×640（符合YOLO标准）

---

### 2. 高性能异构双模型并行推理架构
设计并实现**标志检测+通用目标检测**的融合推断框架：

#### 双模型设计：
- **模型A** (`bestvvv.pt`)：专属交通标志检测
  - 高精度标志识别（停止标志、方向标志等）
  - 自适应多尺度特征提取
  
- **模型B** (`yolo11n.pt` 或 `yolov8n.pt`)：通用目标检测
  - 行人识别（class 0）
  - 车辆识别（car/bus/truck, classes 2, 5, 7）
  - 多类别同时推理

#### 推理融合逻辑：
```
检测模式选择 → 动态模型调度 → 类别过滤 → 结果融合 → 可视化
  ├─ 仅标志模式      → 加载模型A
  ├─ 仅行人/车辆模式 → 加载模型B  
  └─ 标志+行人模式   → 并行载入两模型，融合结果
```

**关键创新**：通过条件模型加载和类别级别的过滤，减少无关类别推理，提升整体推断效率。

---

### 3. 动态参数调整与多场景自适应
实现了**实时参数调整系统**，支持用户在推理过程中动态修改：

- **置信度阈值**（0.10 - 0.95）
  - 实时调整，即时生效
  - 影响检测灵敏度和误报控制
  
- **IOU 阈值**（0.10 - 0.95）
  - 控制非极大值抑制的严厉程度
  - 适应不同拥挤场景

- **检测模式切换**
  - "仅标志"：提高标志检测准确率
  - "仅行人/车辆"：专注动态目标统计
  - "标志+行人"：场景全覆盖

**意义**：避免预设参数的一刀切，适应多样化的应用场景。

---

### 4. 异步多线程交互架构（PyQt6 QThread）
基于 PyQt6 的多线程架构解决**深度学习高负荷推理与UI响应的冲突**：

#### 架构设计：
```
┌─────────────────────────────────────┐
│         UI 主线程 (主窗口)           │
│  ├─ 拖拽响应                         │
│  ├─ 参数实时调整                     │
│  └─ 结果展示                         │
└──────────────┬──────────────────────┘
               │ 信号 (pyqtSignal)
               ↓
┌──────────────────────────────────────┐
│   DetectionWorker 后台线程            │
│  ├─ 模型推理 (GPU加速)                │
│  ├─ 帧处理 (CPU密集)                  │
│  └─ 统计计算                         │
└──────────────┬──────────────────────┘
               │ 信号返回
               ↓
        UI 更新 (帧显示, 统计)
```

#### QThread + Signal/Slot 机制优势：
- ✅ 模型推理不阻塞UI（即使在高分辨率视频下）
- ✅ 流畅的实时预览和参数反馈
- ✅ 安全的线程间通信（避免GIL竞争）
- ✅ 帧率稳定（30+ FPS 摄像头输入）

## 🔧 解决的主要技术问题

### 问题1：UI卡顿 vs 深度学习推理
**根本原因**：模型推理（CPU/GPU密集）与UI事件循环在同一线程，导致界面冻结

**解决方案**：
- 使用 PyQt6 QThread 将推理隔离到后台线程
- 通过 pyqtSignal 实现线程间的信号通信
- 保证主线程专注于UI响应

**效果**：即使在4K视频、实时摄像头下，UI始终流畅响应，FPS稳定 ≥30

---

### 问题2：单一模型的检测局限性
**根本原因**：通用YOLO模型无法准确识别中国交通标志（特有设计），且推理效率低

**解决方案**：
- 训练专属标志模型 (`bestvvv.pt`)，针对CCTSDB2021数据集优化
- 设计双模型架构：标志专用 + 通用目标检测
- 实现条件加载和动态切换，避免无关模型推理

**效果**：标志检测精度提升，总推理延迟降低，支持多场景灵活切换

---

### 问题3：参数一刀切导致的泛化不足
**根本原因**：固定的置信度/IOU阈值无法适应不同场景（低光条件、拥挤场景等）

**解决方案**：
- 实现**动态参数调整系统**（滑块 + 数值输入）
- 支持实时修改，无需重启推理
- 记录每次检测的参数，便于后续分析

**效果**：用户可根据场景快速调优，同一模型适应多种环境

---

### 问题4：缺乏用户管理和数据溯源
**根本原因**：毕设演示需要展现系统化设计，缺失权限管理显得不专业

**解决方案**：
- 实现完整的注册/登录系统
- 采用PBKDF2-SHA256强加密
- 生成Token用于会话管理和数据溯源
- CSV导出时包含用户信息和脱敏Token

**效果**：系统可追踪每次检测的操作者和时间，提升专业度
